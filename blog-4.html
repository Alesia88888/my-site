Project Management Triangle

Project Management Triangle or as it is also called an iron triangle represents constrains, which are time, cost and scope. Each of these constraints is dependable on each other and it is impossible to make changes to one of them without affecting the other.

Time is a period within which we shall complete the project and deliver working software to the customer.

Cost is project’s budget.

Scope is what needs to be done or a list of project deliverables.

Let’s see some examples of how triangle works.

Constraints are always competing with each other. If we need to increase scope, it will lead to increase of time and budget. If we would like to decrease time or budget, it will affect a scope in a way that we’ll have to remove some of deliverables.

The way how we deal with project management triangle may also vary from a methodology that has been selected.

For instance, in Waterfall methodology the main goal for project manager is to keep initially planned time, scope and budget unchanged.

In Agile unchanged shall be time and budget, while scope is not completely defined in the beginning of the project and there definitely will be some changes.
